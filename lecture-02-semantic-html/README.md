# Lecture 02 – Semantic HTML & Structure

## 1. What I implemented this lecture

- I created a semantic portfolio page using professional HTML structure.
- I added a header, navigation, portfolio section, interests section, and footer.
- I included images, project links, and an embedded video using semantic elements.

## 2. Semantic decisions I made (REQUIRED)

### Decision 1

- Element(s) used: `<header>`
- Where in the page: At the top of the page
- Why this element is semantically correct:
  I used `<header>` because it introduces the page and presents my name and professional background. It is the correct semantic element for introductory page content.

### Decision 2

- Element(s) used: `<section>` and `<article>`
- Where in the page: In the main content area, especially in the portfolio section
- Why this element is semantically correct:
  I used `<section>` to group related content under the portfolio topic, and `<article>` for individual portfolio entries because each project or item can stand as its own meaningful content unit.

### Decision 3

- Element(s) used: `<figure>` and `<figcaption>`
- Where in the page: Around images and the embedded video
- Why this element is semantically correct:
  I used `<figure>` for media content that belongs together with a description, and `<figcaption>` to provide a clear explanation of the media for readers and accessibility tools.

---

## 3. Accessibility considerations

- I added a skip link so keyboard and screen reader users can go directly to the main content.
- I used one clear `<h1>` and a logical heading hierarchy with `<h2>` and `<h3>`.
- I added meaningful alt text for all images.
- I used semantic elements such as `<nav>`, `<main>`, `<section>`, `<article>`, and `<footer>` to improve readability and accessibility.

---

## 4. What I learned

- I learned that semantic HTML improves both structure and accessibility.
- I learned that HTML should describe meaning, not appearance.
- I learned how professional page structure supports maintainability and usability.
